Problema de la Mochila 0/1

Seleccionar objetos maximizando el valor total sin exceder la capacidad disponible.

Modelo Matemático

max Σ pᵢxᵢ Σ wᵢxᵢ ≤ M xᵢ ∈ {0,1}

Algoritmos

  • Greedy
  • Backtracking
  • Recocido Simulado

Complejidades

  • Greedy → O(n log n)
  • Backtracking → O(2ⁿ)
  • Recocido Simulado → Aproximado

Ejemplo editable

valores = [60,100,120] pesos = [10,20,30] capacidad = 50 from heuristicos_pkg.knapsack import mochila_greedy seleccion, valor = mochila_greedy(valores,pesos,capacidad) print(seleccion) print(valor)

Puedes modificar libremente valores, pesos y capacidad para probar nuevas instancias.

Problema del Agente Viajero

Este problema pertenece a la clase de problemas NP, lo que significa que no se conoce un algoritmo que lo resuelva en tiempo polinomial para todas las instancias. Por esta razón, los enfoques prácticos recurren a métodos heurísticos y metaheurísticos que, sin garantizar optimalidad, producen soluciones de buena calidad en tiempos aceptables. El Problema del Agente Viajero (TSP) es uno de los problemas de optimización combinatoria más estudiados en la investigación operativa y la ciencia de la computación. Dado un conjunto de ciudades y las distancias entre cada par de ellas, el TSP busca encontrar la ruta más corta que visite cada ciudad exactamente una vez y regrese al punto de partida.

Modelo Matemático

C(x) = \sum_{k=1}^{n-1} d_{x_k x_{k+1}} + d_{x_n x_1}

Algoritmos

  • Generación Lexicográfica de Permutaciones (PERMLEXDESENRUTAR)
  • Búsqueda Local 2-OPT (ASCENSOEMPINADO2OPT)
  • Recocido Simulado para TSP (extensión propuesta)

Complejidades

  • Generación Lexicográfica de Permutaciones (PERMLEXDESENRUTAR) → O(n)
  • Búsqueda Local 2-OPT (ASCENSOEMPINADO2OPT) → O(k*n^3)
  • Recocido Simulado para TSP (extensión propuesta) → (n > 1000)

Pruebas Unitarias

  • Validación de factibilidad
  • Correctitud del backtracking
  • Consistencia del recocido simulado
